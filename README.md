# 暖線拼布工作室｜商用示範網站

「暖線拼布工作室」是一個虛構的拼布／手作課程品牌。本專案用來展示一頁式商用網站可以如何整理個人工作室的品牌介紹、課程、作品、信任內容與預約入口。

> 本網站為商用網站設計示範案例，品牌及部分內容為情境展示用途。

公開展示：<https://gavin1424.github.io/warm-thread-patchwork-demo/>

## 技術

- 原生 HTML、CSS、JavaScript
- 無框架、無後端、無資料庫
- 無需安裝套件或執行建置指令
- 可直接部署到 GitHub Pages 或其他靜態網站服務

## 本機瀏覽

最簡單的方式是直接開啟 `index.html`。若電腦已有 Python，也可在此資料夾啟動本機靜態伺服器：

```powershell
python -m http.server 4173
```

再開啟：

```text
http://127.0.0.1:4173/
```

## 主要檔案

- `index.html`：完整一頁式網站內容、SEO 與結構化資料。
- `styles.css`：版面、響應式設計、色彩與自製布料作品圖形。
- `site-config.js`：品牌、SEO、課程、作品、FAQ、聯絡設定與示範模式。
- `script.js`：內容渲染、手機選單、FAQ、示範聯絡提示與表單行為。
- `assets/hero-patchwork.jpg`：本示範專案全新產生的主視覺，無沿用既有客戶圖片。
- `favicon.svg`：簡易品牌圖示。
- `robots.txt`、`sitemap.xml`：搜尋引擎基礎設定。
- `404.html`：GitHub Pages 找不到頁面時的替代頁。
- `網站資料替換指南.md`：轉換為真實客戶網站的步驟。
- `網站品質檢查報告.md`：本機與公開版本的檢查紀錄。

## 示範模式

`site-config.js` 中的 `DEMO_MODE` 預設為 `true`：

- LINE、預約、電話、社群與網站服務按鈕不會前往假帳號。
- 表單不要求姓名、電話或 Email。
- 表單不會儲存或傳送選擇內容。
- 點擊後會清楚顯示示範案例提示。

正式使用前，必須先填入真實聯絡資料、完成隱私權告知與功能測試，再將 `DEMO_MODE` 改為 `false`。

## 圖片與素材

主視覺是為本示範專案全新產生的情境圖片；作品展示目前使用程式內建的抽象布料圖形，不包含既有客戶素材。正式使用時請替換為客戶自有或已取得合法授權的圖片。

## 部署

本專案所有公開檔案都在根目錄，GitHub Pages 可直接從預設分支根目錄部署。部署前請依實際網址同步更新：

- `index.html` 的 canonical、Open Graph URL 與圖片網址
- `site-config.js` 的 `seo.baseUrl`
- `robots.txt` 的 Sitemap URL
- `sitemap.xml` 的 `<loc>`
- `404.html` 的專案路徑

## 正式上線前最低檢查

1. 替換品牌、課程、作品與聯絡資料。
2. 確認圖片授權、尺寸與替代文字。
3. 驗證所有 CTA 連到正確 LINE、電話或表單。
4. 加入符合實際服務的隱私權與個資說明。
5. 在手機、平板、桌面重新執行品質檢查。
