# 網站資料替換指南

這份指南用來把「暖線拼布工作室」成交展示版快速替換成真實客戶網站。請保留一份乾淨模板，再於新案件資料夾建立副本；不要把客戶資料寫回共用模板。

## 1. 建立獨立客戶副本

1. 複製整個模板到客戶的案件資料夾。
2. 以「品牌名稱_網站_v1」命名副本。
3. 確認副本沒有 `.git`、舊部署設定或其他客戶資料。
4. 修改前取得客戶對文字、圖片、Logo 與聯絡資料的使用授權。
5. 保留本模板的回復點，避免直接覆蓋原始版本。

## 2. 集中替換 `site-config.js`

### 品牌資料

修改 `brand`：

- `name`：品牌名稱
- `englishName`：英文名稱
- `headline`：首頁主標題
- `subheadline`：首頁副標題
- `description`：品牌定位與服務摘要

### 品牌聯絡資料

修改 `contact`：

- `lineUrl`：正式 LINE 加好友或聊天連結
- `phone`：真實電話
- `email`：真實收件信箱
- `instagramUrl`：正式 Instagram 網址
- `facebookUrl`：正式 Facebook 網址
- `formEndpoint`：有完整個資告知的詢問表單網址

`DEMO_MODE: true` 只攔截具有 `data-brand-contact` 的品牌課程按鈕。正式啟用前應：

1. 填入客戶確認的真實聯絡資料。
2. 補上符合實際流程的隱私權與個資說明。
3. 測試每個按鈕與表單。
4. 確認沒有假資料後，再將 `DEMO_MODE` 改為 `false`。

### 網站製作服務資料

修改 `service`：

- `name`：網站製作服務名稱
- `headline`、`subheadline`：服務頁主副標題
- `servicePageUrl`：服務頁相對路徑
- `contactUrl`：主要聯絡入口，建議使用 LINE 或單一詢問頁
- `email`：可公開的服務 Email
- `publicSocialUrl`：可公開的社群帳號
- `inquiryFormUrl`：客戶詢問表單
- `privacyContact`：隱私權聯絡窗口
- `formalPrice`、`casePrice`：正式售價與案例優惠價

網站服務按鈕使用 `data-service-contact`，不受品牌 `DEMO_MODE` 影響。若沒有有效的 `contactUrl`、表單或 Email，按鈕只會顯示安全提示，不會前往假網址。

### SEO 資料

修改 `seo`：

- `baseUrl`：正式公開網址，結尾保留 `/`
- `pages.home`：品牌首頁的 title、description、robots、路徑、分享圖與 JSON-LD
- `pages.service`：網站服務頁的相同資料

`script.js` 會集中更新 title、description、robots、canonical、Open Graph、Twitter Card 與結構化資料。HTML `<head>` 仍保留靜態預設值；正式交付時也要同步替換，確保停用 JavaScript 時資料仍正確。

示範頁目前為 `noindex, follow`。轉為真實營業品牌前，必須先確認品牌真實性、正式聯絡方式與隱私權內容，才評估改成 `index, follow`。

## 3. 替換課程與 FAQ

修改 `courses` 陣列：

- `order`：顯示順序
- `name`：課程名稱
- `summary`：一句課程介紹
- `audience`：適合對象
- `feature`：課程特色
- `duration`：預估時間
- `note`：費用或洽詢說明

修改 `faqs` 陣列中的 `question` 與 `answer`。所有時數、材料、取消、補課、包班與名額規則都必須由客戶確認，不保留不適用的示範假設。

## 4. 替換作品圖片

修改 `works` 陣列：

- `name`：作品名稱
- `category`：作品類型
- `description`：作品重點
- `image`：圖片相對路徑，例如 `assets/works/bag-01.webp`
- `alt`：描述看得見的作品，不堆疊關鍵字
- `visualClass`：圖片失敗時的 CSS 備援圖形

圖片規格建議：

- 統一 4:3。
- 單張 900–1200px 寬。
- 優先使用 WebP 或經壓縮的 JPEG。
- 檔名使用小寫英文、數字與連字號。
- 不使用來源不明、未授權、假冒學員作品或含其他客戶資料的圖片。

載入失敗時網站會回到 `visualClass` 指定的 CSS 圖形。換成真實客戶圖片後，刪除首頁的「情境示範素材」聲明；若仍混用生成圖，必須保留清楚標示。

## 5. 替換固定文案

`index.html` 內需要逐項確認：

- 導覽名稱
- 工作室介紹
- 適合哪些人
- 上課流程
- 教學特色
- 學習體驗內容
- 預約區
- 頁尾
- 示範案例與作品素材聲明
- 隱私權內容

不得使用假姓名、假照片、假星級、假學生數量、假營業年份、假證照或假媒體報導。真實學員回饋必須先取得公開授權。

`service.html` 內需要逐項確認：

- 適合對象與客戶問題
- 交付成果與託管說明
- 免費子網域與自有網域差異
- 包含／不包含範圍
- 製作期、修改次數與付款方式
- 價格、優惠資格與 FAQ
- 正式聯絡與隱私權內容

價格或範圍修改時，必須同步核對正式方案文件，避免銷售頁與報價不一致。

## 6. 替換品牌顏色

主要色彩集中在 `site-config.js > theme`：

- `ink`：主要文字與深色背景
- `paper`：淺色區塊背景
- `cream`：網站主背景
- `clay`：主要品牌色與 CTA
- `clayDark`：品牌色深色版
- `olive`、`blue`：輔助色

改色後重新檢查一般文字、按鈕、頁尾、深色區塊與鍵盤焦點的對比。

## 7. 更新網址與搜尋設定

正式網址確定後，更新：

1. `site-config.js > seo.baseUrl` 與頁面路徑。
2. `index.html`、`service.html` 的靜態 canonical、Open Graph 與 JSON-LD 預設值。
3. `robots.txt` 的規則與 Sitemap URL。
4. `sitemap.xml` 的 `<loc>` 與 `<lastmod>`。
5. `404.html` 的首頁路徑。
6. 部署後用瀏覽器確認動態 SEO 最終值。

## 8. 正式交付前檢查

- 所有文字均為客戶確認版本。
- 所有圖片可合法商用且有正確 alt。
- 沒有其他客戶資料、本機絕對路徑或內部開發備註。
- 390px、768px、1440px 沒有橫向溢位或文字截斷。
- 手機選單、CTA、FAQ、表單與隱私權入口可用鍵盤操作。
- 品牌按鈕與網站服務按鈕的行為彼此獨立。
- 公開首頁與 `service.html` 回傳 HTTP 200 且能互相返回。
- title、description、robots、canonical、Open Graph、Twitter Card 與 JSON-LD 正確。
- 最終版本有 Git commit 或其他可回復備份。
