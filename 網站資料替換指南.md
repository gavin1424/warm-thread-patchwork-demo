# 網站資料替換指南

這份指南用來把「暖線拼布工作室」示範站快速替換成真實客戶網站。請保留一份未修改的模板，再於新案件資料夾建立副本，不要直接把客戶資料寫回共用模板。

## 1. 建立客戶專案副本

1. 複製整個網站資料夾到該客戶的案件資料夾。
2. 以「品牌名稱_網站_v1」命名副本。
3. 確認副本沒有 `.git`、舊部署設定或其他客戶資料。
4. 修改前先取得客戶對文字、圖片、Logo 與聯絡資料的使用授權。

## 2. 集中替換 `site-config.js`

大部分會反覆變動的資料都集中在 `site-config.js`：

### 品牌資料

修改 `brand`：

- `name`：品牌名稱
- `englishName`：英文名稱，沒有可留空並同步調整 HTML 字標
- `headline`：首頁主標題
- `subheadline`：首頁副標題
- `description`：品牌定位與服務摘要

### SEO 資料

修改 `seo`：

- `title`：搜尋結果標題，建議包含品牌與核心服務
- `description`：約 70–110 個繁體中文字，清楚說明服務與對象
- `baseUrl`：正式公開網址，結尾保留 `/`
- `ogImage`：社群分享圖檔路徑

修改後，也要同步更新 `index.html` 的 `<title>`、description、canonical、Open Graph、Twitter Card 與 JSON-LD。不要只改 JavaScript 設定而漏掉 HTML SEO 標籤。

### 課程資料

修改 `courses` 陣列。每堂課包含：

- `order`：顯示順序
- `name`：課程名稱
- `summary`：一句課程介紹
- `audience`：適合對象
- `feature`：課程特色
- `duration`：預估時間
- `note`：費用或洽詢說明

可以新增或刪除課程，但建議一頁式網站先保留 3–6 個最重要方案。

### 作品資料與圖片

修改 `works` 陣列：

- `name`：作品名稱
- `category`：作品類型
- `description`：作品重點
- `image`：圖片相對路徑，例如 `assets/works/bag-01.webp`
- `alt`：圖片替代文字，描述看得見的作品，不堆疊關鍵字
- `visualClass`：圖片無法載入時使用的備援圖形樣式

圖片建議：

- 相同比例，建議 4:3。
- 單張寬度約 900–1200px。
- 優先使用 WebP 或經壓縮的 JPEG。
- 檔名使用小寫英文、數字與連字號。
- 不使用來源不明、未授權或含其他客戶私人資訊的圖片。

若 `image` 保持空字串，網站會繼續使用自製抽象布料圖形；填入有效路徑後會自動顯示實拍作品，載入失敗時仍會回到備援圖形。

### 常見問題

修改 `faqs` 陣列中的 `question` 與 `answer`。回答必須符合客戶實際規則，尤其是時數、材料、取消、補課、包班與名額，不要保留示範假設。

## 3. 替換聯絡與社群資料

所有聯絡資料集中在 `site-config.js` 的 `contact`：

- `lineUrl`：正式 LINE 加好友或聊天連結
- `phone`：真實電話
- `email`：真實收件信箱
- `instagramUrl`：正式 Instagram 網址
- `facebookUrl`：正式 Facebook 網址
- `formEndpoint`：已確認安全且有個資告知的表單端點

目前 `DEMO_MODE: true`。正式串接前保持此設定，避免按鈕誤連或假裝送出。

正式啟用步驟：

1. 填入真實且經客戶確認的聯絡資料。
2. 依需要調整 `script.js` 的聯絡按鈕行為。
3. 補上完整隱私權與個資蒐集告知。
4. 測試 LINE、電話、Email、社群與表單。
5. 確認沒有假資料後，將 `DEMO_MODE` 改為 `false`。

若使用第三方表單，不要把私密 API Key 寫入前端檔案。

## 4. 替換品牌顏色

主要顏色集中在 `site-config.js` 的 `theme`，載入時會套用到 CSS 變數：

- `ink`：主要文字與深色背景
- `paper`：淺色區塊背景
- `cream`：網站主背景
- `clay`：主要品牌色與 CTA
- `clayDark`：主要品牌色深色版
- `olive`、`blue`：輔助色

改色後應檢查文字與背景對比。CTA、頁尾、深色區塊及焦點外框不能因配色改動而難以辨識。

## 5. 替換頁面固定文案

下列內容在 `index.html`：

- 導覽名稱
- 工作室介紹
- 適合哪些人
- 上課流程
- 教學特色
- 學習體驗情境文案
- 預約區說明
- 頁尾網站製作服務提示
- 示範案例聲明
- 隱私權說明

正式網站必須刪除或改寫所有「示範」「情境」「待串接」文字。若使用真實學員回饋，請先取得公開授權，不使用假姓名、假照片或假星級。

## 6. 更新搜尋與分享資訊

正式網址確定後，更新：

1. `index.html` 的 canonical URL。
2. `index.html` 的 `og:url` 與 `og:image`。
3. `index.html` 的 JSON-LD 名稱、網址與描述。
4. `site-config.js` 的 `seo.baseUrl`。
5. `robots.txt` 的 Sitemap URL。
6. `sitemap.xml` 的 `<loc>` 與 `<lastmod>`。
7. `404.html` 的 CSS、favicon 與首頁路徑。

## 7. 正式交付前檢查

- 所有文字均為客戶確認版本。
- 所有圖片可合法商用且有正確 alt。
- 沒有其他客戶資料、本機絕對路徑或內部開發備註。
- 手機 390px、平板與桌面沒有橫向溢位。
- 手機選單、所有 CTA、FAQ、表單與隱私權入口可操作。
- Demo 模式切換符合實際聯絡功能。
- 公開網址回傳 HTTP 200。
- title、description、Open Graph、robots、sitemap 與 404 均已更新。
- 最終版本已備份，並提供客戶基本操作與交付說明。
